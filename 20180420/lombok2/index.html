lombok之注解使用

  • lombok
  • 2018-04-20

一.lombok对象与注解

  lombok 通过注解和定义对象来简化代码的书写.比如val 用于定义局部finnal变量.var 用于定义局部动态变量.@NonNull 进行非空判断.

二.demo的依赖包

  使用lombok时除了要使用lombok插件外还需要添加lombok的依赖

三.val 局部finally变量

   利用val替代实际final变量的声明,让lombok自动根据赋值表达式来替换val.替换的好处就是不需要自己判定赋值表达式返回的数据类型,当然源代码长度也缩减了. lombok的目的也就是为了减少冗余代码.

import lombok.val;
import java.util.ArrayList;
/**
 * @author yp
 * @time 2018/4/13
 * @description  可以用val替代局部final变量的实际声明.由val代替的声明
 */
public class TestVal {
    public static void main(String[] args){
        val list = new ArrayList<String>();
        //等同于ArrayList<String> list = new ArrayList<String>();
        list.add("0");
        list.add("1");
        System.out.println(list.get(0));
        list = new ArrayList<String>();
    }
}

四.var 局部变量

   var这个对象其实目前还处于实验阶段,所以在使用var时需要注意,需要现在lombok.config中声明:

lombok.var.flagUsage = ALLOW

   这样lombok插件才会去读取由var标注的变量,并进行转换.

import lombok.experimental.var;
import java.util.ArrayList;
/**
 * @author yp
 * @time 2018/4/16
 * @description
 */
public class TestVar {
    public static void main(String[] args){
        var s = new ArrayList<String>();
        s.add("aaa");
        s.add("bbb");
        System.out.println("==>"+s.get(0));
        s = new ArrayList<String>();
        s.add("cccc");
        System.out.println("==>"+s.get(0));
    }
}

   这里需要注意的是,请将lombok.config放在src下,而不是resources下,编译优于合并.

五.@NonNull 注解

   在构造方法或者放入入参时,在所有方法体代码内的顶层进行对参数进行null判断(但是,super()和this()会比它先执行).
import lombok.NonNull;

/**
 * @author yp
 * @time 2018/4/20
 * @description
 */
public class TestNonNull extends Something{

    public TestNonNull(@NonNull String s) {
        super("Hello1");
        System.out.println(s);
    }

    private TestNonNull(Integer s) {
        super("Hello2");
        if(s == null){
            throw new NullPointerException("normal");
        }
        System.out.println(s);
    }

    public static void main(String[] args){
        String s = null;
        TestNonNull t1 = new TestNonNull(s);
        Integer a = null;
        TestNonNull t2 = new TestNonNull(a);
    }
}

/**
 * @author yp
 * @time 2018/4/20
 * @description
 */
public class Something {
    public Something(String s){
        System.out.println(s);
    }

    public Something(Integer s){
        System.out.println(s);
    }
}


从代码和上图可以看出,@NonNull 是在构造方法的super()方法之后进行null检查的,跟if(s == null)效果一致
